Roadmap: Roda do Sabor

Fase 1: Refatoração, Modularização e Áudio

Estruturação base para escalabilidade e correção de bugs core.

CONCLUÍDO

  • Isolar Banco de Dados (comidas.js e sons.js)

    Separação do objeto de comidas, receitas e implementação das listas de áudio independentes para limpeza do app.js.

  • Correção do Sintetizador e Autoplay

    Desbloqueio do `AudioContext` no clique do usuário para garantir que sons nativos toquem em todos os navegadores móveis.

  • Sistema de Áudio em 3 Fases (Suspense)

    Implementação do fluxo: Som de Giro ➔ Som de Parada ➔ Pausa de 1s ➔ Confetes + Som de Vitória.

  • Painel de Testes (Debug Tool)

    Criação do `temp.html` para manipulação rápida de moedas, hard resets e limpeza do localStorage durante desenvolvimento.

Fase 2: Integração Cloud (Firebase)

Remover a dependência exclusiva do localStorage e salvar progresso na nuvem.

EM ANDAMENTO

  • Configurar Firebase & Autenticação Anônima

    Criar o projeto no Google Firebase e ativar o login "sem senha" para atrelar os dados ao ID invisível do dispositivo do usuário.

  • Cloud Firestore: Progresso do Usuário

    Migrar a lógica do `core.js` para salvar/ler as Moedas, Temas e Sons desbloqueados diretamente do banco de dados.

  • Cloud Firestore: Logs de Comidas Personalizadas

    Criar rotina que captura toda vez que o usuário adiciona uma "Nova Comida" e envia para uma coleção global chamada `logs_comidas`.

Atenção ao Modo Offline: Como o app é um PWA, precisamos configurar o Firebase Firestore com `enablePersistence()`. Isso garante que se o usuário estiver no 4G ruim, o app continue funcionando e envie as comidas manuais pro banco assim que a internet voltar.

Fase 3: Imersão e Segurança (UX/UI)

Polimento das animações, tato (haptics) e proteção de dados.

PENDENTE

  • Sanitização de Inputs (Segurança XSS)

    Criar função para limpar textos digitados no input de "Nova Comida", bloqueando inserção de tags <script> antes de salvar no DB.

  • Feedback Tátil (Vibration API)

    Programar pequenos pulsos (`navigator.vibrate(50)`) enquanto a roleta gira e uma vibração comemorativa forte ao final.

  • Transições CSS Suaves

    Remover a aparição "dura" (display: flex repentino) do modal de vitória, adicionando fade-in (opacity) e scale (zoom).

Nota sobre iOS: A API de vibração nativa da web tem suporte muito limitado em iPhones (Safari). Precisamos colocar os comandos de vibração dentro de um `try...catch` para o app não quebrar caso o celular não suporte a função.

Fase 4: Recursos Sociais e Administrativos

Página de análise de métricas para o dono do app e expansão social.

PENDENTE

  • Página Secreta de Admin (`admin.html`)

    Criar uma página com senha restrita ao criador do app, que lê e exibe em formato de tabela todas as comidas da coleção `logs_comidas` do Firebase.

  • Web Share API (WhatsApp)

    Adicionar botão no modal de vitória: "Compartilhar Resultado". Irá gerar uma mensagem automática: "A Roda do Sabor decidiu que hoje vamos comer 🍕 Pizza! Vamos?"

  • Update Helper do Service Worker

    Aprimorar o `sw.js` para detectar novas versões do app automaticamente, avisando o usuário para atualizar e limpando o cache velho do celular.

Segurança Admin: O arquivo `admin.html` não deve ser referenciado em nenhum link dentro do `index.html`. Você só acessará digitando a URL direta.
